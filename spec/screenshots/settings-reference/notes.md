# Linear settings reference notes

Captured with bundled Browser Use from the logged-in in-app browser session. Each PNG is a visible viewport screenshot, and each matching .dom.txt file contains the DOM snapshot used to confirm the page had loaded before capture.

Captured pages: 18

Key patterns observed:

- Settings uses a dedicated left sidebar with Back to app at top.
- Account pages are under /settings/account/* and include Preferences, Profile, Notifications, Security & access, Connected accounts, and Agent personalization.
- Issues, Projects, Features, and Administration sections appear as grouped settings nav blocks.
- Preferences is mostly card rows with label/description left and dropdown/toggle/button right.
- Most deeper pages can be convincingly represented as polished Tier 2 list/card pages.
